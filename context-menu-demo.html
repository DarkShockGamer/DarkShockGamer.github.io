Reusable context menu component for desktop and mobile

Overview

This custom context menu component replaces the default browser context menu with a styled, context-aware menu that works on both desktop (right-click) and mobile (long-press).

Features

  • Context-aware menu items (different options for images, links, and text)
  • Desktop right-click support
  • Mobile long-press detection with haptic feedback
  • Keyboard navigation (Arrow keys, Enter, Escape)
  • ARIA attributes for accessibility
  • Matches site's design using CSS variables
  • Easy to extend with custom menu items
  • Toast notifications for user feedback

Integration

Step 1: Include the Files

Add these lines to your HTML <head> section:

<!-- Custom Context Menu --> <link rel="stylesheet" href="/assets/css/context-menu.css"> <script src="/assets/js/context-menu.js"></script>

Step 2: That's It!

The context menu automatically initializes when the page loads. No additional configuration is needed for basic usage.

Step 3: Customize (Optional)

You can customize the menu items by modifying the configuration:

API Reference

ContextMenu.config

Configuration object with the following properties:

  • longPressDuration - Duration in ms for long-press detection (default: 500)
  • preventDefaultContextMenu - Prevent browser's default context menu (default: true)
  • imageMenuItems - Array of menu items for images
  • linkMenuItems - Array of menu items for links
  • textMenuItems - Array of menu items for text selection
  • defaultMenuItems - Array of default menu items

ContextMenu.showToast(message, duration)

Display a toast notification:

ContextMenu.showToast('Action completed!', 2000);

ContextMenu.close()

Programmatically close the context menu:

ContextMenu.close();

Menu Item Structure

Each menu item is an object with these properties:

{ label: 'Menu Item Label', // Required: Text to display icon: 'üìã', // Optional: Emoji or text icon action: (target) => { // Required: Function to execute // target is the element that was clicked/long-pressed console.log('Action executed', target); } }

Accessibility

  • Keyboard navigation with Arrow keys
  • Enter or Space to select menu item
  • Escape to close menu
  • ARIA roles and attributes for screen readers
  • Focus indicators for keyboard users
  • Respects prefers-reduced-motion
  • High contrast mode support

Browser Support

The context menu works on all modern browsers including:

  • Chrome/Edge (latest)
  • Firefox (latest)
  • Safari (latest)
  • Mobile browsers (iOS Safari, Chrome Mobile)

Note: Some features like clipboard API require HTTPS or localhost.

Troubleshooting

Menu not appearing on mobile

Make sure you're holding your press for at least 0.5 seconds. The device may vibrate when the long-press is detected (if vibration is supported).

Clipboard operations not working

The Clipboard API requires a secure context (HTTPS or localhost). If you're testing on HTTP, these features won't work.

Custom menu items not showing

Make sure your custom configuration is applied after the DOM is loaded and before the first context menu interaction.
